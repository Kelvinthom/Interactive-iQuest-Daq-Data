iQuest-Vib-Daq Data Specification

The iQuest-Vib-Daq system provides high-frequency vibration and condition monitoring data for predictive analytics and mechanical fault detection.

1. Data Streams

The Data Acquisition (DAQ) system segments data into logical streams for optimized ingestion.

DAQ_Info (Metadata)

Static metadata describing the physical environment and sensor mapping.

  • Location: Physical or operational site (e.g., "Plant-A-Section-4").
  • assetID: Unique UUID or Tag Name of the machine.
  • sensorLabels: Array of positions (e.g., MTR-NDE-H).
  • Units: Array of measurement units used (g, mm/s, °C, etc).
{
  "DAQ_Info": {
    "Location": "BM-Primary-Milling",
    "assetID": "Ball-mill-3",
    "sensorLabels": ["MTR-NDE-H", "G/OU-DE-V", "MTR-DE-H"],
    "Units": ["g", "mm/s", "degrees C"]
  }
}

S_aRMS_t & S_vRMS_t

RMS Trends: Acceleration and Velocity Root Mean Square values. These are the primary indicators of machine health severity according to ISO 10816 standards.

S_aTW & S_vTW

Time Waveforms: High-resolution time-domain samples. Critical for detecting "impacting" in bearings or gear teeth cracks that FFT might smooth out.

S_aFFT & S_vFFT

Spectral Data: Fast Fourier Transform results. Breaks down vibration into frequency components to identify specific failure modes like unbalance, misalignment, or looseness.

S_temp & speedRMS

Operational Context: Temperature and Shaft Speed. Used to correlate vibration increases with load or thermal expansion.
